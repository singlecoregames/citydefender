# City Defender — Game Design Document

> Missile Command × Active Incremental. 수동 요격으로 시작해, 점점 자동 포탑 군단을
> 지휘하는 사령관이 되어가는 4–5시간 분량의 인디 게임.
>
> 레퍼런스: **Missile Command**(코어 플레이), **Shelldiver**(진행 구조 — 능동형
> incremental, 분기형 스킬트리, 명확한 엔딩), **SNKRX**(아트 — 네온 도형 + 블룸).

---

## 1. 핵심 컨셉 (Core Pillars)

1. **Missile Command를 그대로**: 떨어지는 미사일을 클릭/탭 지점 폭발로 요격, 도시를 지킨다.
2. **수동 → 자동으로의 전이**: 초반엔 100% 수동 플레이. 중반부터 자동 포탑이 해금되며,
   후반엔 자동화가 주력이 되고 수동 사격은 강력한 보너스(+@)가 된다.
3. **능동형 incremental**: Shelldiver처럼 AFK가 아닌, 계속 플레이하며 4–5시간 안에
   엔딩을 보는 구조. 방치 요소는 최소화.
4. **읽히는 미니멀리즘**: SNKRX식 네온 도형. 모든 비주얼은 절차적으로 그려서
   외부 아트 에셋 없이 완성한다.

## 2. 게임 루프

```
┌─────────────────────────────────────────────────────┐
│  NIGHT (런, 2~5분)                                   │
│  웨이브 방어 → 적 격추로 Scrap 획득 → 밤 종료        │
│  (도시 생존 수에 따라 보너스, 전멸 시 조기 종료)      │
└───────────────┬─────────────────────────────────────┘
                ▼
┌─────────────────────────────────────────────────────┐
│  DAY (메타, 30초~2분)                                │
│  스킬트리에 Scrap/Cores/Data 투자                    │
│  도시 수리, 포탑 배치/교체, 다음 밤 준비             │
└───────────────┬─────────────────────────────────────┘
                ▼ 다음 NIGHT (더 강한 웨이브)
```

- **Night(밤) = 1 런.** 밤 `n`은 `4 + floor(n/3)` 웨이브로 구성 (초반 2분 → 후반 5분).
- 도시가 전부 파괴되면 밤이 조기 종료되고 해당 밤 수익의 60%만 획득 (소프트 실패 — 진행은 항상 전진).
- 같은 밤을 재도전하는 게 아니라 **밤 번호는 계속 전진**하되, 스킬트리로 "시작 밤 선택"을
  해금하면 낮은 밤으로 내려가 farm 가능 (Shelldiver의 expedition 커스터마이즈에 대응).
- **엔딩**: Night 50 "The Swarm" 보스 밤 클리어 시 엔딩 연출. 이후 자유 플레이(Endless) 개방.

## 3. 코어 게임플레이 (Missile Command 충실 재현 + 변형)

### 3.1 플레이어 측
- 화면 하단에 **도시**(시작 3개, 최대 6개)와 **포대 슬롯**(시작 1개: 중앙 수동 캐논, 최대 5개).
- **수동 캐논**: 클릭/탭한 지점으로 요격 미사일 발사 → 도달 지점에서 폭발(원형, 시간에 따라
  팽창 후 소멸). 폭발 반경 내 적 미사일에 데미지.
- **탄약**: 클래식 MC의 웨이브당 고정 탄약 대신, **재생형 탄창** (기본 6발, 1.5초당 1발 재생).
  업그레이드로 탄창/재생속도 강화. "탄약 관리" 긴장감은 유지하되 incremental답게 성장.
- 도시는 HP를 가짐 (기본 1히트 파괴 → 업그레이드로 HP/실드/자동수리).

### 3.2 적 유닛 (도형 언어)
| 적 | 도형 | 등장 | 행동 | HP |
|---|---|---|---|---|
| Ballistic | 가는 선 + 점 헤드 | Night 1 | 상단에서 직선 낙하 | 1 |
| Heavy | 굵은 선 + 큰 헤드 | Night 5 | 느리지만 단단함 | 3+ |
| MIRV | 다이아몬드 헤드 | Night 8 | 중고도에서 3~5갈래 분열 | 2 |
| Cruise | 가로 삼각형 | Night 12 | 사인 곡선 횡단 후 급강하 | 2 |
| Smart Bomb | 깜빡이는 원 | Night 18 | 폭발 가장자리를 회피 기동 | 1 |
| Bomber | 큰 사각형 | Night 22 | 횡단하며 폭탄 투하 | 8+ |
| Shielded | 육각형 외곽선 | Night 30 | 실드(첫 폭발 무효) | 3 |
| **Boss** | 거대 복합 도형 | Night 10/20/30/40/50 | 패턴 보스 (아래 3.4) | 다량 |

적 HP/속도/수량은 밤 번호에 따라 지수 스케일 (`HP × 1.10^n`, 수량 `× 1.06^n` — 밸런스 시뮬로 튜닝).

### 3.3 수동 플레이의 후반 가치 (+@ 설계)
자동화가 깡DPS를 담당하더라도 수동이 계속 의미 있도록:
- **Overcharge Shot**: 수동 사격은 항상 "현재 전체 포탑 DPS의 X%"를 추가 데미지로 갖는
  버스트 — 자동화가 강해질수록 수동 한 방도 같이 강해짐.
- **Combo Meter**: 수동 요격 연속 성공 시 전역 데미지/수익 배율 증가 (놓치면 감소).
- **Active Abilities**(스킬트리 해금): EMP(전체 슬로우), Mega Bomb(거대 폭발), Time Dilation
  (3초 슬로모) — 전부 수동 발동, 쿨다운제.
- 보스의 약점 코어는 수동 사격으로만 노출되는 패턴 1개씩 포함.

### 3.4 보스 (5회)
- **N10 Carrier**: 대형 사각형, MIRV 연속 사출. 파괴 시 첫 Core 다량.
- **N20 Hydra**: 분열 반복하는 다이아몬드 군체.
- **N30 Bastion**: 실드 재생 — 수동 Overcharge로 실드 브레이크 필요.
- **N40 Eclipse**: 화면 절반을 가리는 장막 + 장막 뒤 발사.
- **N50 The Swarm**: 전 적 유형 총동원 + 3페이즈. 클리어 = 엔딩.

## 4. Incremental 시스템

### 4.1 화폐 3종 (Shelldiver의 다중 화폐 점진 해금 구조)
| 화폐 | 획득 | 용도 | 해금 |
|---|---|---|---|
| **Scrap** (⬡ 노랑) | 적 격추, 밤 완료 보너스, 생존 도시당 보너스 | 스킬트리 대부분의 노드 | 처음부터 |
| **Cores** (◆ 시안) | 보스, 밤 첫 클리어, 도전 과제성 마일스톤 | 대형 해금 노드(포탑 종류, 포대 슬롯, 도시 슬롯, 어빌리티) | Night 10 보스 |
| **Data** (▣ 마젠타) | "퍼펙트 웨이브"(피해 0), 콤보 유지, 수동 요격 비율 | 자동화 지능 트리(타게팅 AI, 우선순위, 효율) | Night 20 이후 |

Data를 "잘한 플레이"에서만 주는 것이 핵심 — 자동화 게임이 되어도 능동 플레이가 보상받는 축.

### 4.2 스킬트리 (5개 브랜치, 약 100노드)
중앙에서 5방향으로 뻗는 분기형 트리 (Shelldiver식 — 원하는 가지를 골라 빌드 커스터마이즈).

```
                [Cannon 18노드]
                      │
 [Economy 20노드] ─ [CORE] ─ [Automation 26노드]
                      │
        [City 16노드] │ [Tech/Special 20노드]
```

**① Cannon (수동, 18노드)** — Scrap 중심
- 폭발 반경 +15%×5, 데미지 ×1.5×5, 탄창 +2×4, 재생속도, 더블샷(1클릭 2발),
  체인 폭발(요격된 적이 소폭발), Overcharge % 강화, 크리티컬.

**② Automation (26노드)** — Cores로 해금, Scrap/Data로 강화
- 포대 슬롯 +1 ×4 (◆)
- 포탑 5종 해금 (◆): **Gatling**(단일 연사) → **Flak**(범위 산탄) → **Laser**(지속 빔)
  → **Missile Pod**(유도탄) → **Railgun**(관통 저격)
- 종별 데미지/사거리/연사 노드, 전역 발사속도, "포탑도 콤보 적용".

**③ City (16노드)**
- 도시 슬롯 +1 ×3 (◆), 도시 HP, 실드(밤당 1회 무효), 자동 수리, 생존 보너스 강화,
  "폐허 수익"(파괴된 도시도 50% 수익).

**④ Economy (20노드)**
- Scrap 획득 ×1.25 ×6, 밤 완료 보너스, Salvage Drone(낙하 잔해 자동 수집 → 보너스 Scrap),
  이자(보유 Scrap의 %를 밤마다), 퍼펙트 웨이브 보상 ×2, Data 획득량.

**⑤ Tech/Special (20노드)**
- Active Ability 3종 해금 (◆) + 쿨다운/위력 강화
- **시작 밤 선택** (낮은 밤 farm 허용), **웨이브 스킵**(쉬운 웨이브 자동 정산),
  슬로모 진입, 타게팅 AI 모드 (Data): 근접 우선 / 도시 위협 우선 / 고HP 우선 / 스마트(혼합).

### 4.3 비용/수익 곡선 (초기값 — 시뮬레이터로 튜닝)
- 노드 비용: 브랜치별 `base × 1.35^(브랜치 내 구매 수)`. base: Cannon 10, City 15, Economy 20, Automation 25(Scrap 노드).
- 격추 보상: `tier보상 × 1.12^night × (1 + 콤보보너스)`.
- 밤 완료 보너스: `25 × 1.15^night × (생존 도시/전체 도시)`.
- 목표 페이싱 검증은 **헤드리스 밸런스 시뮬레이터**(§구현 플랜)로 자동화한다.

### 4.4 4–5시간 페이싱 로드맵
| 구간 | 플레이타임 | 상태 |
|---|---|---|
| Phase 1 — 수동의 시대 | 0–30분 (N1–9) | 수동 캐논만. 탄약 관리 긴장감. Cannon/Economy 첫 노드들 |
| Phase 2 — 첫 포탑 | 30–90분 (N10–19) | 첫 보스 → Cores → Gatling 해금. "내가 안 쏴도 죽는다"는 첫 쾌감 |
| Phase 3 — 사령관 | 1.5–3시간 (N20–34) | 포탑 3~4종 + 타게팅 AI + 어빌리티. 수동은 Overcharge/콤보 역할로 전환 |
| Phase 4 — 최적화 | 3–4.5시간 (N35–49) | 풀 빌드 완성, 시작 밤 farm, 트리 마무리 |
| Phase 5 — 결전 | 4.5–5시간 (N50) | The Swarm → 엔딩 → Endless 개방 |

## 5. 아트 디렉션 (SNKRX 스타일 — 픽셀아트 + 은은한 글로우)

핵심은 **"매끄러운 벡터 네온"이 아니라 "깔끔한 픽셀아트 도형"**. SNKRX처럼
어두운 체크무늬 바닥 위에 또렷한 픽셀 도형이 놓이고, 글로우는 거들 뿐 주인공이 아니다.

- **픽셀화 파이프라인**: Three.js를 저해상도 버퍼(`RESOLUTION ≈ 360p`)에 렌더한 뒤
  CSS `image-rendering: pixelated`로 nearest-neighbor 업스케일 → 모든 도형이 굵은
  픽셀 그리드에 스냅. `antialias: false`. (이 한 줄 상수로 픽셀 크기 조절.)
- **배경**: 거의 같은 명도의 두 **무채색** 회색(`#262626`/`#1f1f1f`) **체크무늬 바닥**
  + 밝은 회색(`#404040`) 아레나 프레임(바닥선/양쪽 벽). 2×2 픽셀 텍스처를 nearest로
  타일링 (외부 에셋 0).
- **글로우는 은은하게**: `UnrealBloomPass` 강도 0.55 / threshold 0.55 — 밝은 코어만
  살짝 발광. 과한 네온 금지.
- **팔레트**: SNKRX 원작의 7색을 그대로 사용 — fg `#dadada` · yellow `#facf00` ·
  orange `#f07021` · red `#e91d39` · green `#8bbf40` · blue `#019bd6` ·
  purple `#8e559e`. UI(DOM)와 스킬트리도 같은 팔레트로 통일.
- **도형 언어**: SNKRX처럼 **도형이 진영을 말한다** — 플레이어측 유닛/탄환 = 원(스네이크),
  적 = 둥근 사각형(스플리터는 45° 다이아몬드, 보스는 천천히 회전), 구조물(도시/캐논/
  지원 건물) = 둥근 블록. 색은 양 진영이 같은 팔레트를 공유.
- **드롭 섀도**: SNKRX처럼 **모든 스프라이트가 우하단으로 하드 섀도**를 떨어뜨린다
  (월드 기준 고정 오프셋 ≈ 렌더 4px, 반투명 검정 — 아래 깔린 대상 위에도 자연스럽게
  드리움). DOM UI도 글로우 대신 같은 방향의 오프셋 섀도로 통일.
- **타격 피드백**: 피격 시 흰색 플래시(0.09s), 스폰 시 오버슈트 팝인, 사망 시 유닛
  색 파티클 팝, 트레일은 유닛 색의 어두운 톤.
- **이펙트**: 폭발 = SNKRX AoE 방식 — **밝은 흰색 additive 채움 원**이 착탄 순간
  최대 반경으로 나타나 잠시 유지된 뒤 **축소되며 소멸**(데미지 반경 로직과 동일:
  즉시 최대 → hold → shrink). 테두리는 **끊어진 호 4개(각 ~70°, 두껍게)**가 서로
  다른 속도·방향으로 빠르게 공전. 운석(적)은 낙하 중 각자 다른 속도로 회전(텀블링),
  미사일 = 짧은 픽셀 트레일, 도시 피격 = 화면 셰이크. 콤보 = 캐논 주변 링 게이지.
- **사운드**: ZzFX(절차적 SFX)로 레트로 톤 통일. BGM은 후순위(앰비언트 1~2트랙).

## 6. UX / 플랫폼별 입력
- **데스크톱**: 마우스 클릭 = 발사. 1/2/3 = 어빌리티. Space = 일시정지. 휠 = (Endless) 배속.
- **모바일**: 탭 = 발사. 하단 어빌리티 버튼. 도시/포대는 화면 최하단에서 약간 띄워 손가락 가림 방지.
- 스킬트리/메뉴는 DOM 오버레이 (캔버스 UI보다 접근성/반응형/i18n에 유리).
- 세이브: 자동 (밤 종료마다 + 30초 주기). 슬롯 1개 + 클라우드(Steam).

## 7. 범위 제외 (v1에서 안 하는 것)
- 멀티플레이, 광고/IAP, 프레스티지 루프(엔딩 후 Endless로 갈음), 외부 아트 에셋,
  모드 지원. — 전부 v1 이후 검토.
